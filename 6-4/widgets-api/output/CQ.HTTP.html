Class CQ.HTTP

Package:CQ
Class:HTTP
Extends:Object
Clientlib:cq.widgets
A helper class providing a set of HTTP-related utilities.
The response object returned by the methods get and post has the following properties:
  • body: String
    The response body
  • headers: Object
    The response headers:
  • Status: Number
    The status code
  • Message: String
    (optional) The status message
  • Location: String
    (optional) The location
  • Path: String
    (optional) The path
You can use the isOk method to verify the response object.

This class is a singleton and cannot be created directly.

Encodes the path of the specified URL if it is not already encoded. Path means the part of the URL before the first question mark or hash sign.
See CQ.shared.HTTP.encodePath for details about the encoding.
Sample:
/x/y+z.png?path=/x/y+z >> /x/y%2Bz.png?path=x/y+z
Note that the sample would not work because the "+" in the request parameter would be interpreted as a space. Parameters must be encoded separately.
