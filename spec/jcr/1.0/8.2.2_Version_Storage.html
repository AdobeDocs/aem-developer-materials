8.2.2 Version Storage

A version history consists of a single nt:versionHistory node with a set of immediate child nodes all of type nt:version, representing all the versions within that version history.

An nt:versionHistory has at least one child, the nt:version node representing the root version. From the root version the version graph proceeds through a network of REFERENCE properties linking any additional child nt:version nodes into a version graph defining the successor relations among the versions.
