3.13.2.4 Version History Reference

Apart from nodes within the same shared set, which share the same version history, each versionable node within a persistent workspace has its own version history.

Under both simple and full versioning the version history of a node is accessed through VersionManager.getVersionHistory, which returns a VersionHistory object (see ยง15.1.1 VersionHistory Object). Under full versioning it is also represented by the REFERENCE property jcr:versionHistory, which points to an nt:versionHistory node (see ยง3.13.5.1 nt:versionHistory).
