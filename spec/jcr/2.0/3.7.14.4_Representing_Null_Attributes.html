3.7.14.4 Representing Null Attributes

The attributes that make up a node type definition may in some cases have no set value (for example, some child node definitions may not define a default primary type). To store this information (i.e., the lack of a value) in an nt:nodeType, nt:childNodeDefinition or nt:propertyDefinition node the property representing that attribute must simply be not present, since null values for single-value properties are not permitted (see ยง10.4.2.4 No Null Values).
