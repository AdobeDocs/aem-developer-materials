3.4.2.1 Parent Resolution

In most cases an item will have only one parent, in such a case, parent resolution is trivial. In repositories that support shareable nodes, a node may share its child nodes and properties with other nodes. A child item of a shared node therefore has more than one parent. In such a case the parent resolved depends upon the deemed path of the item, which is an implementation-specific issue (see ยง3.9.5 Deemed Path). An attempt to resolve the parent of a workspace root node always fails.
