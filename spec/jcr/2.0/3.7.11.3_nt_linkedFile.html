3.7.11.3 nt:linkedFile

[nt:linkedFile] > nt:hierarchyNode primaryitem jcr:content

- jcr:content (REFERENCE) mandatory

The nt:linkedFile node type is similar to nt:file, except that the content node is not stored directly as a child node, but rather is specified by a REFERENCE property. This allows the content node to reside anywhere in the workspace and to be referenced by multiple nt:linkedFile nodes. The content node must be referenceable. Support for this node type requires support for referenceable nodes with referential integrity (see ยง3.8.2 Referential Integrity).
