3.7.1.7 Primary Item

A node type can declare one of its child items as primary, meaning that for all nodes of that type, that child item is accessible through a dedicated API method which does not require the name of the item. (see ยง5.1.7 Primary Item Access). This feature can help generic API clients intelligently traverse an unknown node structure. The primary item may be an item name, which must be a JCR name, or null, meaning that there is no primary item.

3.7.1.7.1 Primary Item and Same-Name Siblings

In cases where the primary child item specifies the name of a set of same-name sibling child nodes, the node with index [1] will be regarded as the primary item.

3.7.1.7.2 Property and Child Node With Same Name

In cases where this node has both a child node and a property with the same name and where that name is specified as the primary item name, the child node will be regarded as the primary item (see ยง22.4 Property and Node with Same Name).
