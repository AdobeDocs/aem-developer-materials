14.1.2 Shared Node Creation on Import

During import the behavior of the IMPORT_UUID_COLLISION_THROW indicates that if an incoming referenceable node has the same identifier as an existing mix:shareable node in the workspace, the incoming node is created and added to the shared set of the existing mix:shareable node (see ยง3.9.1 mix:shareable). Note that if the import in question is a Session import (see ยง11.6.2 Session Event-Based Import and ยง11.7.2 Session Stream-Based Import) new shared transient nodes will be created. These nodes are not considered to be new, in the sense that Node.isNew will return false.
