11.8.4 Throw on Identifier Collision

IMPORT_UUID_COLLISION_THROW: If an incoming non-shareable node has the same identifier as a node already existing in the workspace, then either a SAXException is thrown by the ContentHandler (in the case of event-based import) or an ItemExistsException is thrown by the importXML method (in the case of stream-based import). In the case of shareable nodes, the behavior differs (see ยง14.1.2 Shared Node Creation on Import).
